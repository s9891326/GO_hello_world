# 第七章練習題

## 練習 1：學生管理系統

創建一個完整的學生管理系統：

要求：
- 設計 Student 結構體（姓名、學號、年齡、專業、成績等）
- 設計 Course 結構體（課程名稱、學分、教師等）
- 設計 Grade 結構體（學生、課程、分數、學期等）
- 實現相關方法：添加課程、計算 GPA、查詢成績等
- 使用構造函數模式創建學生

## 練習 2：圖書館管理系統

設計圖書館管理系統的數據結構：

要求：
- 設計 Book 結構體（書名、作者、ISBN、分類等）
- 設計 Member 結構體（會員信息、借閱歷史等）
- 設計 Loan 結構體（借閱記錄）
- 實現借書、還書、查詢等方法
- 使用結構體嵌入實現不同類型的會員（普通會員、VIP會員等）

## 練習 3：員工薪資計算系統

創建員工薪資管理系統：

要求：
- 設計基本 Employee 結構體
- 使用嵌入創建不同類型員工（全職、兼職、合約工）
- 實現薪資計算方法（不同員工類型有不同計算方式）
- 添加獎金、扣款功能
- 生成薪資報表

## 練習 4：電商產品管理

設計電商平台的產品管理系統：

要求：
- 設計 Product 結構體（包含 JSON 標籤）
- 設計 Category、Brand、Review 等相關結構體
- 實現產品搜索、過濾功能
- 支援 JSON 序列化和反序列化
- 實現產品評分計算

## 練習 5：配置管理系統

創建應用程序配置管理系統：

要求：
- 設計 Config 結構體支援多種配置選項
- 使用選項模式實現靈活的配置創建
- 支援從 JSON 文件加載配置
- 實現配置驗證功能
- 支援配置的熱重載

## 練習 6：社交媒體用戶系統

設計社交媒體的用戶系統：

要求：
- 設計 User 結構體（用戶信息、隱私設置等）
- 設計 Post 結構體（帖子內容、點讚、評論等）
- 設計 Follow 關係結構體
- 實現用戶註冊、發帖、關注等功能
- 使用建造者模式創建複雜的帖子

## 提示

1. 合理設計結構體字段，考慮零值友好性
2. 選擇合適的方法接收者（值 vs 指針）
3. 使用組合而非繼承的思想
4. 適當使用結構體標籤
5. 實現適當的構造函數和驗證邏輯
6. 考慮內存對齊和性能
